Create the i18n foundation that all other tasks depend on.

## Files to create

### `i18n/config.js`
```js
export const locales = ['en', 'es'];
export const defaultLocale = 'en';
```

### `app/contexts/LocaleContext.js`
- React Context providing the current locale string
- `LocaleProvider` component that accepts `locale` prop (from URL segment)
- `useLocale()` hook
- Sets `document.documentElement.lang` via useEffect

### `i18n/useTranslations.js`
- `useT()` hook that returns the UI dictionary for the current locale
- Static imports of `i18n/ui/en.json` and `i18n/ui/es.json`
- Falls back to English if locale not found

### `i18n/useContent.js`
- Hooks: `useCards()`, `useAstrology()`, `usePlanets()`, `useSigns()`, `useCourt()`
- Each hook reads locale from context, returns the matching content module
- English content: imports from original `utils/*.js` files (unchanged)
- Spanish content: imports from `i18n/content/es/*.js` (created in Qw0l)
- Falls back to English if locale not found

## Design notes
- All hooks are client-side (`'use client'`)
- No new npm dependencies
- The `utils/` files are NOT modified — they remain the English source of truth
- Non-translatable data (symbols, date ranges, element keys) still imports from `utils/` directly
