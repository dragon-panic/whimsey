Move all pages under a `[locale]` dynamic segment so URLs become `/en/signs`, `/es/court`, etc.

## Changes

### 1. New minimal root layout: `app/layout.js`
- Just `<html>` + `<body>`, imports `globals.css`
- No Navigation, no providers — those move to the locale layout

### 2. New root redirect page: `app/page.js`
- `'use client'` component
- Checks `localStorage.getItem('whimsey-locale')`
- Redirects to `/${savedLocale || 'en'}` via `router.replace()`
- Renders null (or a loading spinner)

### 3. Move current layout → `app/[locale]/layout.js`
- Receives `params.locale` from the URL segment
- Wraps children in `LocaleProvider` (from ROuH) and `DeckProvider`
- Renders `Navigation`
- Exports `generateStaticParams` returning `[{locale:'en'},{locale:'es'}]`
- Exports `generateMetadata` with locale-aware title/description

### 4. Move pages
- `app/page.js` (Decan) → `app/[locale]/page.js`
- `app/court/page.js` → `app/[locale]/court/page.js`
- `app/signs/page.js` → `app/[locale]/signs/page.js`
- `app/planets/page.js` → `app/[locale]/planets/page.js`

### 5. Fix imports
- All moved files need updated relative import paths (e.g., `../utils/` → `../../utils/`)
- CSS module imports need path updates too

## Notes
- `basePath: '/whimsey'` in next.config.js stays unchanged — Next.js auto-prepends it
- `generateStaticParams` in layout.js covers all child pages
- Static export will produce: `out/index.html`, `out/en/`, `out/es/` with all subpages
- Hash navigation (#aries-1, #saturn) is unaffected — hashes are locale-independent data keys
